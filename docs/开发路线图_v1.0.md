## 0. 现状回顾

原型阶段已验证三件最关键的事:

1. **FIFO 配对可用** —— 从交割单流水能干净重建每笔完整交易(1:1 配对)。
2. **确定性指标可信** —— 胜率、盈亏比、持有周期、成本、处置效应等全部由代码算出。
3. **策略感知诊断成立** —— 叠加行情后,能识别打板接力并算出连板高度胜率、接力成功率、核按钮/溢价兑现纪律、情绪周期适应性等专属指标。

产品 1.0 = 把原型工程化为一个**用户能自助上传交割单、拿到可信诊断报告**的最小可用产品。

---

## 1. 1.0 范围(做什么 / 不做什么)

### 1.0 必做
- 交割单上传与解析:覆盖 **3–5 种主流来源**(同花顺、通达信、及 2–3 家头部券商导出格式)。
- 计算内核:FIFO 配对 + A股成本规则 + 通用行为指标(从原型抽成独立模块)。
- 交易风格自动识别:至少区分 **打板接力 / 低吸 / 趋势波段 / 价投** 四类(置信度 + 兜底为通用)。
- 策略感知诊断:通用标尺 + **打板接力专属标尺**(已有),其余风格先用通用标尺。
- AI 叙述层:指标 → 大模型 → 人话诊断与建议(带事实校验,严格合规约束)。
- 报告:在线查看 + 导出 PDF;含健康分、KPI、行为画像、逐项诊断、整改清单。
- 隐私与合规:脱敏 + 加密 + 用完即删 + 隐私政策 + 合规文案审查。

### 1.0 暂不做(留待 1.x / 2.0)
- 实盘对接、自动交易;历史趋势/多账户对比;社区与训练营;打板以外风格的专属深度标尺;分钟/tick 级分析;付费墙(先免费跑量验证留存)。

## 3. 模块任务拆解(原型 → 生产)

### 3.1 解析层 `parser`(护城河,投入最大)
- 抽象统一接口:输入文件 → 标准成交流水 `[{时间,代码,名称,方向,价格,数量,费用...}]`。
- 为每种来源写一个适配器(字段映射 + 编码/分隔符/表头容错)。
- 支持 Excel/CSV/PDF;PDF 走表格抽取。
- 解析失败时收集样本、引导用户反馈,快速扩展覆盖。
- 起点:`prototypes/*/` 里的 `csv.DictReader` 读写逻辑。

### 3.2 计算内核 `fifo` / `cost` / `metrics`
- 把 `analyze.py`、`engine_dabp.py` 内的 FIFO 段与成本函数抽成纯函数模块,补单元测试(配对守恒、成本规则对齐、边界:拆单/部分成交/同股多次)。
- A股微观规则做对:T+1、印花税卖出单边、佣金最低 5 元、过户费、涨跌停无法成交、停牌。

### 3.3 风格识别器 `style_classifier`(1.0 新增重点)
- 输入完整交易集,输出风格标签 + 置信度。
- 打板接力判据(原型已用雏形):打板入场占比高、平均持有 ≈1 个交易日、买入价贴近当日涨停价、单标的重仓、高频切换。
- 低吸/趋势/价投各自的特征(持有周期分布、入场相对均线位置、换手率、集中度)。
- 低置信度兜底为「通用」标尺,避免错诊。

### 3.4 行情接入 `market_loader`
- 日线 OHLC + 涨停价 + 连板数;情绪温度(涨停家数/连板梯队/晋级率)需全市场数据。
- 选型:商用数据源或合规的公开数据;注意**数据版权,不转售受限源**。
- 按「代码+日期」与交易集 join(原型 `engine_dabp.py` 已实现 join 逻辑)。

### 3.5 AI 叙述层 `narrator`
- 输入 `metrics.json` → Prompt 模板 → 大模型 → 每个问题的「现象/证据/危害/建议」四段式。
- **硬约束**:数字只能引用 metrics 里的值(做后校验,发现编造则拦截);禁止荐股/预测/收益承诺。
- 文案风格:通俗、具体、可执行;按识别出的风格切换话术。

### 3.6 报告与应用层
- 复用原型的单文件 HTML + Chart.js 渲染;增加 PDF 导出。
- Web:上传 → 解析进度 → 报告;账户体系最小化。

## 4. 风险与应对

| 风险 | 应对 |
|------|------|
| 多格式解析覆盖不全 | M2 聚焦主流格式 + 失败反馈通道快速迭代 |
| 风格误判导致错诊 | 置信度阈值 + 低置信兜底通用标尺 |
| AI 编造数字/越界荐股 | 两层架构 + 数字后校验 + 合规 Prompt + 输出审查 |
| 行情数据版权/成本 | 选合规源、明确授权范围,情绪指标可分阶段上线 |
| 隐私顾虑导致不敢上传 | 本地解析优先 + 脱敏加密 + 用完即删 + 透明政策 |
