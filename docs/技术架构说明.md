# 技术架构说明

## 1. 设计总原则

**两层分离:确定性计算层 + AI 叙述层。**

- **确定性计算层**(代码):解析交割单 → FIFO 配对 → 计算全部指标。所有数字都来自这里。
- **AI 叙述层**(大模型):接收已算好的结构化指标,生成人话解读与个性化建议。**不负责算数,不得编造数字。**

这是产品可信度的根基,也是与「把交割单直接丢给大模型」的劣质做法的本质区别。

---

## 2. 分层架构

```
┌─────────────────────────────────────────────────────────────┐
│  应用层  Web 上传 / 报告渲染 / 图表 / 导出 / 账户            │
├─────────────────────────────────────────────────────────────┤
│  AI 叙述层  结构化指标 → Prompt 模板 → 大模型 → 解读+建议     │
│            (严格约束:不编造数字、不荐股、不预测)            │
├─────────────────────────────────────────────────────────────┤
│  诊断层  通用行为指标  +  策略感知指标(按风格路由)          │
│         风格识别器:打板接力 / 低吸 / 趋势 / 波段 / 价投       │
├─────────────────────────────────────────────────────────────┤
│  计算内核  FIFO 配对引擎 · A股微观规则(T+1/印花税/佣金)     │
├─────────────────────────────────────────────────────────────┤
│  数据层  交割单解析(多券商/多格式) · 行情数据接入(可选)   │
├─────────────────────────────────────────────────────────────┤
│  安全层  脱敏 · 加密 · 用完即删 · 隐私政策                    │
└─────────────────────────────────────────────────────────────┘
```

## 5. 关键技术决策

1. **解析层是真正的护城河,不是 AI。** 各券商/同花顺/通达信导出格式差异巨大,「随手上传任意格式都能正确解析」是核心壁垒。
2. **指标确定性化,AI 模板化。** AI 只填解读,数字一律来自计算层并做事实校验,杜绝幻觉。
3. **策略感知诊断。** 先识别风格,再套对应标尺——对打板选手「持有短」是纪律不是毛病。错误的通用诊断会给出错误建议。
4. **隐私前置。** 优先本地/前端解析,降低用户上传顾虑;敏感字段脱敏。
5. **报告单文件化。** HTML 单文件 + CDN 图表库,便于分享、导出 PDF、嵌入。
